# Contributing

## License requirement

By submitting a contribution, you agree that it is licensed under the GNU Affero General Public License v3.0 or later (AGPL-3.0-or-later). Modified versions must retain the same license and make their corresponding source code available when distributed or used to provide a network service.

## Required change documentation

Every contribution and published fork must include a clear description of its changes. At minimum, document:

- the user-facing behavior changed;
- configuration, security, or migration impact;
- tests run and their result;
- known limitations or follow-up work.

Use a pull request description, release notes, or a `CHANGELOG.md` entry. Keep the description with the published source code so users can understand the differences from upstream.

## Publishing modified versions

Publish the complete corresponding source code for every distributed modified version in a publicly accessible repository or provide an equally accessible source download. Include build, installation, and configuration instructions needed to run that version.

This policy complements the AGPL. The AGPL is the controlling legal license; this document describes the project's expected publication and documentation practice.
