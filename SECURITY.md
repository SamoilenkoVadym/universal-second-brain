# Security Policy

## Supported versions

Security fixes are applied to the latest version on the `main` branch until the project publishes stable releases.

## Reporting a vulnerability

Do not open a public issue for suspected vulnerabilities, exposed credentials, or unsafe vault access.

Use GitHub's private vulnerability-reporting feature when available for this repository. If it is unavailable, contact the repository owner privately through GitHub and include:

- a clear description and impact assessment;
- reproduction steps or a minimal proof of concept;
- affected version or commit;
- any suggested mitigation.

Please allow reasonable time for investigation and remediation before public disclosure.
